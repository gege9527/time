# 🕐 Time - 自动化任务调度系统

基于 GitHub Actions 的自动化任务执行平台，支持定时签到、消息推送等自动化操作。

## 🌟 功能特性

- ✅ **自动化执行** - 基于 GitHub Actions 实现无人值守自动运行
- 📅 **灵活调度** - 支持自定义 Cron 表达式配置执行时间
- 👥 **多账号支持** - 单次运行可同时处理多个账户任务
- 🔔 **实时通知** - 集成企业微信机器人推送执行结果
- 🛡️ **安全可靠** - 敏感信息通过 GitHub Secrets 安全存储

## 🚀 快速开始

### 1. 准备工作

Fork 本仓库到你的 GitHub 账户

### 2. 配置 Secrets

在仓库 Settings → Secrets and variables → Actions 中添加以下环境变量：

#### 必需配置
| Secret Name | 说明 | 示例格式 |
|------------|------|---------|
| `TOKEN` | 账户认证令牌<br>支持多行配置，每行一个账号 | ```token1<br>token2<br>token3``` |
| `NOTIFY` | 企业微信机器人 webhook key<br>支持多行配置 | ```key1<br>key2<br>key3``` |

### 3. 启用 GitHub Actions

进入 Actions 页面，启用工作流即可开始自动执行。

## ⏰ 定时配置

默认执行时间：每天北京时间 00:10（UTC 16:10）

如需修改执行时间，请编辑 `.github/workflows/run.yml` 文件中的 `cron` 字段：

```yaml
schedule:
  - cron: '10 16 * * *'  # UTC 时间，对应北京时间次日 00:10
```

### Cron 表达式格式说明

```
* * * * *
│ │ │ │ │
│ │ │ │ └── 星期几 (0-6, 0=周日)
│ │ │ └──── 月份 (1-12)
│ │ └────── 日期 (1-31)
│ └──────── 小时 (0-23)
└────────── 分钟 (0-59)
```

### 常用时间配置示例

| 北京时间 | UTC时间 | Cron表达式 | 说明 |
|---------|--------|-----------|------|
| 每天 00:10 | 16:10 | `10 16 * * *` | 默认配置 |
| 每天 08:00 | 00:00 | `0 0 * * *` | 早间执行 |
| 每天 12:00 | 04:00 | `0 4 * * *` | 午间执行 |
| 每小时执行 | - | `0 * * * *` | 每小时整点 |
| 每周一 09:00 | 01:00 | `0 1 * * 1` | 每周一执行 |

## 📊 通知配置

### 企业微信机器人配置

1. 在企业微信群中添加机器人
2. 获取 webhook 地址中的 key 参数
3. 将 key 值添加到 `NOTIFY` Secret 中

### 通知内容格式

```
✅ 签到成功
> 账号: [账号标识]
> 状态: 签到成功
> 时间: 2024-01-01 00:10:00

---

❌ 签到失败
> 账号: [错误信息]
> 状态: 签到失败
> 时间: 2024-01-01 00:10:00
> 详情: [错误详情]
```

## 🔧 高级功能

### 多账号批量处理

支持在同一 Secret 中配置多个账号，每行一个：

```
account_token_1
account_token_2
account_token_3
```

### 手动触发执行

除了定时执行外，还可以：
- 通过 GitHub Actions 页面手动触发
- 推送代码到 main 分支自动触发

### 日志查看

在 Actions 运行记录中可以查看详细的执行日志，包括：
- 执行时间统计
- 成功/失败账号列表
- 错误详情信息

## ⚠️ 安全注意事项

1. **敏感信息安全**
   - TOKEN 和 NOTIFY 包含敏感信息，请勿泄露
   - 建议定期更新认证令牌
   - 不要在代码中硬编码敏感信息

2. **权限控制**
   - 合理设置仓库访问权限
   - 定期审查 Secrets 配置
   - 监控异常执行情况

3. **使用限制**
   - GitHub Actions 有使用配额限制
   - 避免过于频繁的执行调度
   - 注意 API 调用频率限制

## ❓ 常见问题

### Q: 为什么定时任务没有执行？
A: 请检查：
- GitHub Actions 是否已启用
- Cron 表达式是否正确

### Q: 通知没有收到怎么办？
A: 请确认：
- NOTIFY Secret 配置是否正确
- 企业微信机器人是否正常工作
- 查看 Actions 日志中的错误信息

### Q: 如何调试脚本？
A: 可以：
- 通过 workflow_dispatch 手动触发测试
- 查看详细的执行日志
- 在本地环境中运行测试
